# What is a Milestone Offer?

<Info>

A **Milestone Offer** rewards customers when they achieve predefined spending or transaction milestones. Customers become eligible for rewards only after successfully meeting the configured milestone conditions during the offer period.

</Info>

Milestone Offers enable issuers to encourage customer engagement by rewarding customers as they progress towards configured spending or transaction milestones. Multiple milestone levels can be configured, allowing customers to unlock rewards after completing each milestone.

---

## Create a Milestone Offer

To create a new Milestone Offer, begin from the **Manage Offers** page.

### Step 1: Create a New Milestone Offer

On the **Manage Offers** page:

- Navigate to the **CVP Offers** tab.
- Click **+ Create New Offer**.
- Select **Single Offer**.
- The Milestone Offer creation journey is displayed.

<Frame>

![Step 1 - Create a New Milestone Offer](/images/0.1.png)

</Frame>

After selecting **Single Offer**, the **Offer Type** screen is displayed.

<Note>

Single Offers allow you to configure one offer at a time.

</Note>

---

## Step 1: Select Offer Type

After creating a new offer, the **Offer Type** screen is displayed.

Use this page to select the type of offer you want to create.

Select **Milestone Offers** under the **Transaction Data** category.

The available offer categories include:

- All Benefits
- Transaction Data
- EMI Creation
- Card Events
- Display Only

Under **Transaction Data**, you can create:

- Transaction Offers
- Milestone Offers
- Gamified Offers

<Frame>

![Step 1 - Select Offer Type](/images/0.2.png)

</Frame>

After completing the configuration:

- Select **Milestone Offers**.
- Click **Continue** to proceed to the **Outcome Type** configuration.

<Note>

Milestone Offers reward customers after they successfully complete one or more configured milestones.

</Note>

---

## Step 2: Select Outcome Type

After selecting the offer type, the **Outcome Type** screen is displayed.

Use this page to define the reward customers receive after completing the configured milestone.

Select one of the available outcome types.

The available outcome types include:

- Cashback
- Reward
- Voucher
- Smart Tag
- Partner Points

Some outcome types may appear as **Coming Soon** and are not currently available for selection.

<Frame>

![Step 2 - Select Outcome Type](/images/0.3.png)

</Frame>

After completing the configuration:

- Select the required outcome type.
- Click **Continue** to proceed to the **Offer Details** configuration.

---

## Step 3: Configure Offer Details

After selecting the outcome type, the **Offer Details** screen is displayed.

Use this page to configure the basic information required for the Milestone Offer and define where the offer will be applicable.

### Basic Offer Details

Provide the following information:

- **Offer Name** – Enter a unique name to identify the Milestone Offer.
- **Offer Description** – Enter a brief description of the offer.
- **Offer Duration** – Select the start and end date during which the offer remains active.
- **Eligibility Level** – Select how milestone eligibility should be evaluated.
  - **Account Level** – Evaluates eligibility independently for each account.
  - **Customer Level** – Evaluates eligibility across all accounts belonging to the customer.

<Frame>

![Step 3 - Configure Offer Details](/images/0.4.png)

</Frame>

### Offer Applicability

Configure where the Milestone Offer should be available.

Provide the following information:

- **Issuer** – Select the issuing bank.
- **Program Type** – Select the applicable program type.
- **Apply to all current and future programs** – Enable this option to automatically apply the Milestone Offer across all existing and future programs.
- **Program** – Select one or more applicable programs.
- **Enable progress transfer on variant upgrade** – Enable this option to retain milestone progress when an eligible customer upgrades to another supported card variant.

<Frame>

![Step 3 - Configure Offer Applicability](/images/0.4.1.png)

</Frame>

After completing the configuration:

- Review the configured offer details.
- Click **Continue** to proceed to the **Offer User Base** configuration.

<Note>

If **Apply to all current and future programs** is enabled, the Milestone Offer is automatically associated with all existing and future programs for the selected Program Type.

</Note>

---

## Step 4: Configure Offer User Base

After configuring the **Offer Details**, the **Offer User Base** screen is displayed.

Use this page to define which customers are eligible to participate in the Milestone Offer.

### Select the User Base Type

Choose the appropriate user base type based on your campaign requirements.

The following user base options are available:

| User Base Type | Description |
|----------------|-------------|
| **Static Base** | Uses a fixed list of eligible customers throughout the offer period. |
| **Dynamic Config** | Continuously updates eligible customers based on the configured eligibility criteria until the offer becomes active. |

### Configure Eligibility

After selecting the user base type, configure the eligibility criteria used to identify eligible customers.

The available eligibility options include:

- Account Enrollment
- Opt-In
- Smart Tag Attachment
- Cycle-Wise Enrollment

<Frame>

![Step 4 - Configure Offer User Base](/images/0.5.png)

</Frame>

After completing the configuration:

- Configure the required eligibility criteria.
- Click **Estimate User Base** to calculate the estimated number of eligible customers.
- Review the estimated user count.
- Click **Continue** to proceed to the **Milestone Eligibility** configuration.

<Note>

The configured eligibility determines which customers are allowed to participate in the Milestone Offer.

</Note>

---
### Step 5: Configure the Budget

After configuring the **Offer Calculation**, the **Budget** screen is displayed.

Use this page to allocate the budget for the offer and define the approval workflow required before the offer is published.

#### Configure the Budget

Provide the following information:

- **Set Budget** – Enter the total budget value allocated for the offer.
- **Approver** – Select the user responsible for reviewing and approving the configured budget.

<Info>
The offer cannot be published until the selected approver signs off on the configured budget.
</Info>

![Step 5 - Configure Budget](/images/j.png)

#### Advanced Settings

Expand **Advanced Settings** to configure additional budget-related parameters, if required.

After completing the configuration:

- Review the configured budget and approver details.
- Click **Continue** to proceed to the **Usage Limits** configuration.

---

### Step 6: Configure Posting

After configuring the **Usage Limits**, the **Posting** screen is displayed.

Use this page to define when the offer benefit should be posted to the customer's account and specify the eligibility criteria for posting.

#### Enable Posting

Enable **Posting** to allow the configured benefit to be credited to the customer's account.

#### Configure Posting Eligibility

Select the eligibility criteria that must be satisfied before the benefit is posted.

For each posting eligibility condition:

- Select the required eligibility attribute.
- Configure the corresponding values.
- Specify whether the condition is **Allowed** or **Restricted**, as applicable.
- Click **Add more** to configure additional posting eligibility conditions.

#### Configure Offer Posting

Choose when the offer benefit should be posted.

The following posting options are available:

| Posting Option | Description |
|----------------|-------------|
| **Post Immediately** | Credits the benefit immediately after the configured posting conditions are satisfied. |
| **Schedule** | Credits the benefit at a scheduled date and time. |

![Step 6 - Configure Posting](/images/l.png)

#### Configure Narration

Provide the narration that should be recorded in the ledger during posting and reversal.

Configure the following fields:

- **Posting Narration** – Enter the message to be recorded when the benefit is successfully posted.
- **Reversal Narration** – Enter the message to be recorded if the posted benefit is reversed.

![Step 6 - Configure Narration](/images/l.1.png)

After completing the configuration:

- Review the configured posting settings.
- Click **Continue** to proceed to the **Display Elements** configuration.
