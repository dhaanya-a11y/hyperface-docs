## Introduction

This guide explains how to create, configure, and publish Milestone Offers using the Hyperface Dashboard.

Milestone Offers enable issuers to reward customers when they achieve predefined spending or transaction milestones within a specified time period. These offers help drive customer engagement, increase card usage, and encourage targeted spending behavior.

---

## What are Milestone Offers?

A Milestone Offer is a reward mechanism that grants customers a benefit after they reach a specified spending amount or transaction count within a defined eligibility period.

### Example Scenarios

* Spend ₹5,000 within the next 7 days and earn a reward.
* Complete 5 eligible transactions worth more than ₹1,000 each within 7 days and receive cashback.

### Key Components

A Milestone Offer consists of the following configurations:

* Offer Details
* User Base Configuration
* Milestone Eligibility
* Offer Calculation Rules
* Usage Limits
* Posting Configuration
* Display Elements

---

## Create a Milestone Offer

To create a new Milestone Offer:

1. Log in to the Hyperface Dashboard.
2. Navigate to **Portfolio Growth → Offers → Manage Offers**.
3. Click **Create Offer**.
4. Select **Milestone Offer** as the offer type.
5. Complete the configuration journey and publish the offer.

---

## Step 1: Select Offer Type and Outcome

Choose **Milestone Offer** and configure the reward outcome.

### Supported Outcome Types

* Cashback
* Reward Points
* Voucher
* Smart Tag

---

## Step 2: Configure Offer Details

Configure the basic information required for the offer.

### Offer Information

* Offer Name
* Offer Description
* Start Date & Time
* End Date & Time

### Issuer & Program Configuration

* Issuer
* Program Type
* Program Selection
* Apply To All Programs
* Progress Transfer On Variant Upgrade

---

## Step 3: Configure Offer User Base

Define the customer segment eligible for the offer.

### Static Base

Used when the offer is targeted to a predefined customer group.

* Fixed Customer List
* All Accounts In Scope
* Smart Tag Filtering

### Dynamic Configuration

Used when customers can become eligible dynamically.

* Opt-In Enrollment
* Smart Tag Attachment

### Smart Tag Options

* Allowed
* Denied

---

## Step 4: Configure Milestone Eligibility

Define the milestone conditions customers must satisfy.

### Aggregation Types

#### Sum of Transaction Values

Calculates the total eligible spend accumulated by the customer.

#### Count of Transactions

Calculates the total number of eligible transactions completed by the customer.

### Threshold Configuration

Supported conditions include:

* Greater Than
* Greater Than or Equal To
* Less Than
* Equal To

### Subscription Cycle

Available options:

* Custom Cycle
* Billing Month

### Transaction Eligibility Filters

* MCC
* MID
* Merchant Name
* Transaction Value
* Transaction Type
* POS Entry Mode

### Processing Date Options

* Transaction Date
* Posting Date

---

## Step 5: Configure Offer Calculation

Define how rewards are calculated once the milestone is achieved.

### Calculation Rules

* Flat Multiplier
* Fixed Reward
* Percentage Based Reward
* Slab Based Reward

### Additional Controls

* Reward Point Capping
* Transaction Level Cap
* MCC Cap
* MID Cap
* Currency Cap
* Program Cap

### Reversal Configuration

Enable or disable reward reversals based on business requirements.

---

## Step 6: Configure Usage Limits

Usage limits help control reward exposure and budget allocation.

Available limits include:

* Daily Limit
* Weekly Limit
* Monthly Limit
* Lifetime Limit
* Cycle Count Limit

---

## Step 7: Configure Posting

Configure how and when rewards are posted to eligible customers.

### Posting Options

#### Post Immediately

Rewards are posted immediately after successful validation.

#### Scheduled Posting

Rewards are posted after a configured delay.

### Posting Configuration

* Enable Posting
* Posting Eligibility
* Days After
* Event
* Reward Point Expiry
* Posting Narration
* Reversal Narration

## Step 9: Review and Publish

Before publishing, review all configurations carefully.

### Review Checklist

* Offer Details
* User Base Configuration
* Milestone Eligibility
* Offer Calculation Rules
* Usage Limits
* Posting Configuration
* Display Elements

Click **Publish** once all configurations have been validated.

The offer status changes to **Scheduled** and automatically becomes **Active** at the configured start date and time.

---

## Best Practices

* Review all configurations before publishing.
* Use clear and customer-friendly offer descriptions.
* Validate reward calculations before activation.
* Ensure display content aligns with Hyperface branding guidelines.
* Verify Terms & Conditions before making the offer live.
* Test offer eligibility using sample customer scenarios whenever possible.

---

## Notes

* Active offers have limited edit capabilities.
* Eligibility and calculation rules become locked once the offer is live.
* Changes to published offers may require approval based on configured workflows.
