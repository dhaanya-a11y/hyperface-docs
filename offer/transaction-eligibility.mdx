## Configure Transaction Eligibility

After configuring the **Offer User Base**, define the transaction conditions that determine whether a transaction qualifies for the Transactional Offer.

### Configure Transaction Conditions

Select one or more transaction attributes to define the eligibility criteria for the offer.

The following transaction attributes are available:

| Transaction Attribute | Description |
|-----------------------|-------------|
| **MCC** | Evaluates transactions based on the Merchant Category Code (MCC). |
| **MID** | Evaluates transactions based on the Merchant Identifier (MID). |
| **TID** | Evaluates transactions based on the Terminal Identifier (TID). |
| **POS Entry Mode** | Evaluates transactions based on the Point-of-Sale (POS) entry mode. |
| **Transaction Value** | Evaluates transactions based on the transaction amount. |
| **Transaction Type** | Evaluates transactions based on the transaction type. |
| **Sovereignty Indicator** | Evaluates transactions based on the transaction sovereignty indicator. |
| **Transaction Code** | Evaluates transactions based on the transaction code. |
| **Payer VPA** | Evaluates transactions based on the payer Virtual Payment Address (VPA). |
| **Payee VPA** | Evaluates transactions based on the payee Virtual Payment Address (VPA). |

![Step 6 - Configure Transaction Eligibility](/images/h.png)

For each selected transaction attribute:

- Select the appropriate **comparison operator**.
- Enter or select the corresponding **attribute value**.
- Add additional conditions, if required, to build the eligibility criteria.
- Select the **Transaction Posting Date** that should be used for eligibility evaluation.
