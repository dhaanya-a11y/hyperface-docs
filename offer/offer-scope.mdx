Offer Details capture the core information required to create and identify an offer. These details define the offer, its validity period, and its business purpose, making it easier to manage and distinguish offers within the Hyperface Dashboard.

![Step 4 - Configure Basic Offer Details](/images/e.png)

## Fields

| Field | Description | Example |
|-------|-------------|---------|
| **Offer Name** | Enter a unique and meaningful name that clearly identifies the offer in the dashboard. | Fuel Cashback July 2026 |
| **Description** | Provide a brief description explaining the purpose of the offer and the benefit customers will receive. | Earn 5% cashback on eligible fuel transactions. |
| **Offer Type** | Select the type of offer to define how customers become eligible for rewards. Available options include Transactional, Milestone, Merchant, and Gamified Offers. | Transactional Offer |
| **Outcome Type** | Select the reward customers receive after successfully qualifying for the offer. | Cashback |
| **Validity** | Configure the start and end date during which the offer remains active and available to customers. | 01 Jul 2026 – 31 Jul 2026 |
| **Offer Status** | Specifies the current state of the offer. An offer can be saved as a Draft before publishing or activated when it is ready for use. | Active |

#### Offer Applicability

Define where the offer should be applicable.

Provide the following information:

- **Issuer** – Select the issuer for which the offer will be available.
- **Program Type** – Select the applicable program type.
- **Apply to all current and future programs** – Enable this option if the offer should apply to all existing and future programs.
- **Program** – If the above option is disabled, select one or more applicable programs.

After reviewing the configured details, click **Continue** to proceed to the **Offer User Base** configuration.
