Offer Management enables issuers to create, configure, and manage customer reward campaigns through the Hyperface Dashboard. Whether you're creating a **Transactional Offer** or a **Milestone Offer**, every offer follows a structured configuration journey that guides you through each stage of offer creation.

Each step in the workflow captures a specific aspect of the offer. Completing these configurations ensures that the offer is fully defined, validated, and ready to be published.

## Offer Configuration Journey

During the offer creation process, configure the following components:

| Configuration | Description |
|--------------|-------------|
| **Offer Type** | Select the type of offer to be created. |
| **Outcome Type** | Define the reward or benefit customers receive after meeting the offer criteria. |
| **Offer Scope** | Specify the issuer and program where the offer will be applicable. |
| **Offer User Base** | Identify the customers eligible for the offer. |
| **Eligibility** | Configure the conditions that transactions or milestones must satisfy to qualify for the reward. |
| **Offer Calculation** | Define how the reward is calculated. |
| **Budget** | Configure the budget allocated for the offer. |
| **Usage Limits** | Specify how frequently customers can receive the reward. |
| **Posting** | Configure when and how rewards are credited to customers. |
| **Display Elements** | Define the customer-facing content displayed for the offer. |
| **Communication** | Configure the notifications associated with the offer. |

<Note>
The available configuration options may vary depending on the selected **Offer Type** and **Outcome Type**. Complete each step in the configuration journey before publishing the offer.
</Note>
