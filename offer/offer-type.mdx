# Offer Type

## What is an Offer Type?

**Offer Type** defines how customers become eligible to receive a reward, and is the first choice made in the offer creation journey. Each Offer Type supports a specific business objective and determines the eligibility model — a single transaction, accumulated spend, merchant-specific activity, or engagement-based activity — under which rewards are issued.

Hyperface supports four Offer Types: **Transactional**, **Milestone**, **Merchant**, and **Gamified**.

<Note>
The Offer Type selected here determines the configuration options available in the subsequent steps of the offer creation journey — including which eligibility screen (Transaction Eligibility, Milestone Eligibility, and so on) and which Offer Calculation options are shown. Choose the Offer Type that matches the reward behaviour you want before continuing.
</Note>

## Comparing Offer Types

| Consideration | Transactional | Milestone | Merchant | Gamified |
|---|---|---|---|---|
| Reward triggered by | A single qualifying transaction | Reaching a predefined spend or transaction milestone within a period | An eligible purchase at a participating merchant | Completing an interactive challenge or level |
| Evaluation | Each transaction evaluated independently | Progress accumulated until the milestone is achieved | Eligibility based on merchant-specific criteria | Progress tracked through configured levels or activities |
| Best for | Straightforward cashback or reward campaigns | Encouraging sustained spend or engagement over time | Merchant-funded or partner-specific promotions | Engagement-driven, interactive campaigns |

## Transactional Offer

A **Transactional Offer** rewards customers when an individual transaction satisfies the configured eligibility criteria. Each qualifying transaction is evaluated independently, making this Offer Type suitable for cashback and transaction-based reward campaigns.

## Example: Choosing the correct Offer Type

A bank wants to reward customers who spend a total of ₹50,000 across transactions in a calendar month, rather than rewarding any single transaction.

1. Select **Milestone Offer**, since the reward depends on cumulative spend rather than one qualifying transaction.
2. Continue with the remaining setup.

Selecting **Transactional Offer** here would be incorrect — it would reward every qualifying transaction individually instead of tracking progress towards the ₹50,000 target.

## What Happens After Configuration?

After an Offer Type is selected, the dashboard proceeds to the **Outcome Type** screen. The selected Offer Type determines which eligibility and calculation screens appear later in the journey — for example, a Transactional Offer shows **Transaction Eligibility**, while a Milestone Offer shows **Milestone Eligibility**.

## Which Offer Type Should You Select?

- Reward every qualifying transaction on its own → select **Transactional Offer**.
- Reward customers only after they reach a cumulative spend or transaction target → select **Milestone Offer**.
- Reward purchases made specifically at participating merchants → select **Merchant Offer**.
- Reward customers for completing interactive challenges or activities → select **Gamified Offer**.
