## Overview

The **Offer Calculation** section determines **how rewards are calculated and issued** for qualifying transactions. After a transaction meets the configured eligibility criteria, the selected calculation method is applied to determine the reward value.

Choose the calculation method that best aligns with your campaign objective before proceeding with the remaining offer configuration.

The configured **Offer Calculation Rule** determines only **how the reward amount is calculated**. Reward eligibility is determined by the previously configured **Transaction Eligibility** conditions.
</Note>

## Offer Calculation Rules
Select the appropriate **Offer Calculation Rule** based on how the reward should be calculated.
| Calculation Rule | Description | Example |
|------------------|-------------|---------|
| **Flat** | Awards a fixed reward amount for every qualifying transaction, regardless of the transaction value. | ₹100 Cashback |
| **Percentage** | Awards a percentage of the qualifying transaction amount. The reward varies based on the transaction value. | 5% Cashback |
| **Slab** | Awards different reward values based on predefined transaction value ranges or slabs. | ₹50 for ₹500–₹999, ₹100 for ₹1,000–₹1,999 |

### Flat
**When to use it**
Choose **Flat** when every qualifying transaction should receive the same reward, irrespective of the transaction amount.
**Example**
Reward customers with **₹100 cashback** for every qualifying transaction.

### Percentage
**When to use it**
Choose **Percentage** when the reward should be calculated as a percentage of the qualifying transaction amount.
**Example**
Reward customers with **5% cashback** on every qualifying purchase.

### Slab
**When to use it**
Choose **Slab** when different reward values should be issued based on predefined transaction value ranges.
**Example**
- ₹50 cashback for transactions between **₹500–₹999**
- ₹100 cashback for transactions between **₹1,000–₹1,999**

<Tip>
Select the calculation rule that best matches your campaign objective.
- Use **Flat** for fixed rewards.
- Use **Percentage** when rewards should scale with transaction value.
- Use **Slab** when different transaction ranges should receive different rewards.
</Tip>

After selecting a calculation rule:

1. Configure the corresponding reward value.
2. Review the calculated reward before proceeding.

## Skip Cashback Capping (Optional)

Use **Skip Cashback Capping** to exclude cashback limits configured at specific levels. This option allows the configured offer to bypass selected cashback capping rules during reward calculation.

The following capping levels can be excluded:

| Cashback Capping Level | Description |
|-------------------------|-------------|
| **Program Level** | Ignores cashback limits configured at the program level. |
| **Transaction Level** | Ignores cashback limits configured for individual transactions. |
| **MCC** | Ignores cashback limits configured for specific Merchant Category Codes (MCC). |
| **MID** | Ignores cashback limits configured for specific Merchant Identifiers (MID). |
| **Currency Level** | Ignores cashback limits configured for specific currencies. |

<Warning>

Skipping cashback capping allows rewards to bypass the selected capping rules. Enable this option only when the campaign is intended to override existing cashback limits.

</Warning>

## Allow Reversal

Enable **Allow Reversal** if rewards should be reversed whenever the corresponding qualifying transaction is reversed or refunded. This ensures that reward credits remain consistent with the final transaction status.

<Note>

When **Allow Reversal** is enabled, any reward issued for a qualifying transaction can be reversed if the original transaction is subsequently reversed or refunded.

</Note>

---

## What Happens After Configuration?

Once the **Offer Calculation** is configured, every qualifying transaction is evaluated using the selected calculation rule. The calculated reward is then considered for budget validation, posting, and reward distribution based on the remaining offer configuration.

---

## Complete the Configuration

After completing the **Offer Calculation** configuration:

1. Review the configured **Calculation Rule** and reward values.
2. Verify the **Cashback Capping** and **Allow Reversal** settings, if applicable.
3. Click **Continue** to proceed to the **Budget** configuration.
