# Milestone Eligibility

**Milestone Eligibility** (shown on the dashboard as **Set Milestone Eligibility**) defines how customer activity is aggregated, the threshold required to complete the milestone, the cycle over which activity is measured, and which transactions count toward the milestone.

This screen sits inside the **Set-Up Offer** step of the offer creation journey, after **Offer User Base** and before **Offer Calculation**, **Budget**, and **Posting**.

<Note>
Three concepts work together here: **Offer User Base** determines *who* is tracked, **Transaction Eligibility** (below) determines *which of their transactions* count, and **Milestone Eligibility** determines *how that activity accumul is achieved*.
</Note>

### Sum of Transaction Values

Progress is calculated by adding the value of all qualifying transactions. Customers complete the milestone once the cumulative transaction value reaches the configured threshold.

**Example**

If the threshold is **₹10,000**, transactions of ₹2,000, ₹3,000 and ₹5,000 together complete the milestone.

## Example: Quarterly spend milestone

To reward customers who spend over ₹1s each quarter, for 2 consecutivequarters:

1. Set **Aggregate By** to **Sum of Transaction Values**.
2. Set **Define Threshold** to *"Rewa spends are"* **Greater Than**`10000`.
3. Set **Subscription Cycle Period**
4. Set **Number of Cycles** to `2`.

A customer whose eligible spend exceeds ₹10,000 in a given quarter achieves the milestone for that
quarter; progress resets at the startto 2 quarters.
