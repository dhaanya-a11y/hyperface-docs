# What is a Merchant Offer?

<Info>
A **Merchant Offer** rewards customers for making qualifying purchases at participating merchants. Customers become eligible for the configured reward when their transactions satisfy the merchant-specific eligibility criteria defined during offer creation.
</Info>

Merchant Offers enable issuers to run merchant-specific reward campaigns by targeting purchases made at selected merchants. Eligibility can be configured using merchant-related criteria and, optionally, restricted to specific customer segments using the **Offer User Base**.

Once an eligible customer completes a qualifying transaction at a participating merchant, the configured reward is automatically processed according to the offer configuration.

## Create a Merchant Offer

To create a new Merchant Offer, begin from the **Manage Offers** page.

### Step 1: Create a New Merchant Offer

On the **Manage Offers** page:

- Navigate to the **Merchant Offers** tab.
- Review the existing Merchant Offers, if required.
- Click **+ Create New Offer**.
- The Merchant Offer creation journey is displayed.

<Frame>

![Step 1 - Create a Merchant Offer](/images/1.png)

</Frame>

After clicking **+ Create New Offer**, the **Offer Details** screen is displayed.

<Note>

Merchant Offers are created from the **Merchant Offers** tab under **Manage Offers**.

</Note>

## Step 1: Configure Offer Details

After creating a new Merchant Offer, the **Offer Details** screen is displayed.

Use this page to configure the basic information required for the Merchant Offer and define where the offer will be applicable.

### Basic Offer Details

Provide the following information:

- **Offer Name** – Enter a unique name to identify the Merchant Offer.
- **Offer Description** – Enter a brief description of the offer for internal reference.
- **Offer Duration** – Select the start and end date and time during which the offer remains active.
- **Eligibility Level** – Select how the offer eligibility should be evaluated.
  - **Account Level** – Evaluates eligibility independently for each account.
  - **Customer Level** – Evaluates eligibility across all accounts belonging to the customer.

![Step 1 - Configure Offer Details](/images/2.png)

### Offer Applicability

Configure where the Merchant Offer should be available.

Provide the following information:

- **Issuer** – Select the issuing bank for which the Merchant Offer will be applicable.
- **Program Type** – Select the applicable program type.
- **Apply to all current and future programs** – Enable this option to automatically apply the Merchant Offer to all existing and future programs.
- **Program** – If the above option is disabled, select one or more applicable programs.

![Step 1 - Configure Offer Applicability](/images/2.2.png)

After completing the configuration:

- Review the configured offer details.
- Click **Continue** to proceed to the **Offer User Base** configuration.

<Note>

The Merchant Offer is created in **Scheduled** status and automatically becomes **Active** once the configured start date and time is reached.

If **Apply to all current and future programs** is enabled, the offer is automatically associated with all existing and newly created programs for the selected Program Type.

</Note>

## Step 2: Configure Offer User Base

After configuring the **Offer Details**, the **Offer User Base** screen is displayed.

Use this page to define which customers are eligible to participate in the Merchant Offer.

### Select the User Base Type

Choose the appropriate user base type based on your campaign requirements.

The following user base options are available:

| User Base Type | Description |
|----------------|-------------|
| **Static Base** | Uses a fixed list of eligible customers. Once the offer goes live, the user base remains unchanged. |
| **Dynamic Config** | Continuously updates the eligible customer list based on the configured criteria until the offer is published. |

### Configure Eligibility

After selecting the user base type, configure the eligibility criteria used to identify eligible customers.

The following eligibility options are available:

| Eligibility Option | Description |
|--------------------|-------------|
| **Smart Tag** | Identifies eligible customers using the configured Smart Tag. |
| **Accounts** | Defines eligibility using account-related information. |
| **Switch Account Number** | Identifies customers using the Switch Account Number. |
| **Birthdate** | Defines eligibility based on the customer's birth date. |
| **Account Creation Date** | Defines eligibility based on the account creation date. |

![Step 2 - Configure Offer User Base](/images/3.png)

After configuring the required eligibility criteria:

- Click **Estimate User Base** to calculate the estimated number of eligible customers.
- Review the estimated user count.
- Click **Continue** to proceed to the **Budget** configuration.

<Note>

Configuring the Offer User Base is optional.

If no eligibility criteria are configured, the Merchant Offer is available to all customers within the selected program scope.

If a **Smart Tag** is configured, only customers belonging to the selected Smart Tag are eligible to participate in the Merchant Offer.

</Note>

## Step 3: Configure Budget

After configuring the **Offer User Base**, the **Budget** screen is displayed.

Use this page to allocate the budget for the Merchant Offer and define the approval workflow before the offer is published.

### Configure Budget

Provide the following information:

- **Set Budget** – Enter the total budget allocated for the Merchant Offer.
- **Approver** – Select the user responsible for reviewing and approving the configured budget.

![Step 3 - Configure Budget](/images/j.png)

### Advanced Settings

Expand **Advanced Settings** to configure additional budget-related parameters, if required.

After completing the configuration:

- Review the configured budget details.
- Verify the selected approver.
- Click **Continue** to proceed to the **Display Elements** configuration.

<Info>

The Merchant Offer cannot be published until the selected approver approves the configured budget.

</Info>

## Step 4: Configure Display Elements

After configuring the **Budget**, the **Display Elements** screen is displayed.

Use this page to configure how the Merchant Offer is presented to customers in the Progressive Web Application (PWA). The Display Elements configuration consists of two sections:

- **Home Page**
- **Offer Details Page**

Configure both sections to provide a consistent and engaging customer experience.

---

### Configure the Home Page

The **Home** page allows you to define how the Merchant Offer appears on the customer's home screen.

Provide the following information:

- **Enable Display** – Enable this option to display the Merchant Offer in the customer's PWA. Disable it if the offer should remain active but should not be visible to customers.
- **Labels** – Select one or more labels to categorize the Merchant Offer.
- **Display Title** – Enter the title displayed for the Merchant Offer on the PWA.
- **Display Description** – Enter a brief description displayed below the title.
- **Display Order** – Specify the display priority of the Merchant Offer.
- **Display Color** – Select the primary color used for the Merchant Offer card.
- **Offer/Merchant Logo** – Upload the logo displayed for the Merchant Offer.
- **Background Illustration** – Upload the background image displayed on the Merchant Offer card.

<Frame>

![Step 4 - Configure Home Page](/images/4.png)

</Frame>

<Frame>

![Step 4 - Configure Home Page Settings](/images/4.4.png)

</Frame>

After completing the configuration:

- Review the configured Home page settings.
- Click **Continue** to proceed to the **Offer Details Page** configuration.

<Note>

Turn **OFF** the **Enable Display** toggle if the Merchant Offer should remain active in the backend but should not be displayed to customers in the PWA.

</Note>

---

## Step 5: Configure the Offer Details Page

After configuring the **Home Page**, the **Offer Details Page** screen is displayed.

Use this page to configure the detailed information customers see after opening the Merchant Offer in the Progressive Web Application (PWA).

### Section Details

Use the **Section Details** editor to organize and present detailed information about the Merchant Offer.

Provide the following information:

- **Section Title** – Enter a title for the section.
- **Section Content** – Add the section content using the Rich Text Editor (RTE).
- **Add Section** – Click **Add Section** to create additional sections, such as **How It Works**, **Terms & Conditions**, or other offer-related information.
- **Reorder Sections** – Drag and drop sections to change their display order.
- **Delete Section** – Remove an existing section, if required.

### Configure the Call-to-Action (CTA)

Provide the following information:

- **Text On Button** – Enter the text displayed on the Call-to-Action (CTA) button.
- **Redirection Link** – Enter the URL to which customers are redirected after clicking the CTA button.
- **Coupon Code** – Enter the coupon code associated with the Merchant Offer, if applicable.

### Advanced Settings

Enable **Advanced Settings** to configure additional display options for the Merchant Offer, if required.

<Frame>

![Step 5 - Configure Offer Details Page](/images/5.png)

</Frame>

<Frame>

![Step 5 - Configure Offer Details Settings](/images/5.5.png)

</Frame>

After completing the configuration:

- Review the configured Offer Details page settings.
- Click **Continue** to proceed to the **Communication** configuration.

<Note>

Use the Rich Text Editor (RTE) to format the section content with headings, lists, links, and other supported formatting options to improve readability.

</Note>
