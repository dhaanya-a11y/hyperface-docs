## Step 1: Configure Offer Details

After creating a new Merchant Offer, the **Offer Details** screen is displayed.

Use this page to configure the basic information required for the Merchant Offer and define where the offer will be applicable.

### Basic Offer Details

Provide the following information:

- **Offer Name** – Enter a unique name to identify the Merchant Offer.
- **Offer Description** – Enter a brief description of the offer for internal reference.
- **Offer Duration** – Select the start and end date and time during which the offer remains active.
- **Eligibility Level** – Select how the offer eligibility should be evaluated.
  - **Account Level** – Evaluates eligibility independently for each account.
  - **Customer Level** – Evaluates eligibility across all accounts belonging to the customer.

![Step 1 - Configure Offer Details](/images/2.png)

### Offer Applicability

Configure where the Merchant Offer should be available.

Provide the following information:

- **Issuer** – Select the issuing bank for which the Merchant Offer will be applicable.
- **Program Type** – Select the applicable program type.
- **Apply to all current and future programs** – Enable this option to automatically apply the Merchant Offer to all existing and future programs.
- **Program** – If the above option is disabled, select one or more applicable programs.

![Step 1 - Configure Offer Applicability](/images/2.2.png)

After completing the configuration:

- Review the configured offer details.
- Click **Continue** to proceed to the **Offer User Base** configuration.

<Note>

The Merchant Offer is created in **Scheduled** status and automatically becomes **Active** once the configured start date and time is reached.

If **Apply to all current and future programs** is enabled, the offer is automatically associated with all existing and newly created programs for the selected Program Type.

</Note>

## Step 2: Configure Offer User Base

After configuring the **Offer Details**, the **Offer User Base** screen is displayed.

Use this page to define which customers are eligible to participate in the Merchant Offer.

### Select the User Base Type

Choose the appropriate user base type based on your campaign requirements.

The following user base options are available:

| User Base Type | Description |
|----------------|-------------|
| **Static Base** | Uses a fixed list of eligible customers. Once the offer goes live, the user base remains unchanged. |
| **Dynamic Config** | Continuously updates the eligible customer list based on the configured criteria until the offer is published. |

### Configure Eligibility

After selecting the user base type, configure the eligibility criteria used to identify eligible customers.

The following eligibility options are available:

| Eligibility Option | Description |
|--------------------|-------------|
| **Smart Tag** | Identifies eligible customers using the configured Smart Tag. |
| **Accounts** | Defines eligibility using account-related information. |
| **Switch Account Number** | Identifies customers using the Switch Account Number. |
| **Birthdate** | Defines eligibility based on the customer's birth date. |
| **Account Creation Date** | Defines eligibility based on the account creation date. |

![Step 2 - Configure Offer User Base](/images/3.png)

After configuring the required eligibility criteria:

- Click **Estimate User Base** to calculate the estimated number of eligible customers.
- Review the estimated user count.
- Click **Continue** to proceed to the **Budget** configuration.

<Note>

Configuring the Offer User Base is optional.

If no eligibility criteria are configured, the Merchant Offer is available to all customers within the selected program scope.

If a **Smart Tag** is configured, only customers belonging to the selected Smart Tag are eligible to participate in the Merchant Offer.

</Note>

## Step 3: Configure Budget

After configuring the **Offer User Base**, the **Budget** screen is displayed.

Use this page to allocate the budget for the Merchant Offer and define the approval workflow before the offer is published.

### Configure Budget

Provide the following information:

- **Set Budget** – Enter the total budget allocated for the Merchant Offer.
- **Approver** – Select the user responsible for reviewing and approving the configured budget.

![Step 3 - Configure Budget](/images/j.png)

### Advanced Settings

Expand **Advanced Settings** to configure additional budget-related parameters, if required.

After completing the configuration:

- Review the configured budget details.
- Verify the selected approver.
- Click **Continue** to proceed to the **Display Elements** configuration.

<Info>

The Merchant Offer cannot be published until the selected approver approves the configured budget.

</Info>
