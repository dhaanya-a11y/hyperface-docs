## What is a Transactional Offer?

<Info>
A **Transactional Offer** rewards customers for completing a **single qualifying transaction** that satisfies the configured eligibility criteria. Each qualifying transaction is evaluated independently, and eligible customers receive the configured reward based on the selected outcome type.
</Info>
   
---

## Create a Transactional Offer

To create a new Transactional Offer, begin from the **Manage Offers** page.

### Step 1: Create a New Offer

On the **Manage Offers** page:

- Click **+ Create New Offer** located in the upper-right corner.
- From the available options, select **Single Offer**.
- The offer creation journey is displayed.

![Step 1 - Create a Transactional Offer](/images/a.png)

---

### Step 2: Select the Offer Type

After selecting **Single Offer**, the **Offer Type** screen is displayed.

On this screen:

- Browse the available offer types.
- Select **Transaction Offers**.
- Click **Continue**.
- The **Outcome Type** screen is displayed.

![Step 2 - Select the Offer Type](/images/c.png)

---

### Step 3: Select the Outcome Type

After clicking **Continue**, the **Outcome Type** screen is displayed.

The Outcome Type determines the reward that customers receive after completing a qualifying transaction.

On this screen:

- Browse the available outcome types.
- Select the required outcome type.
- Available options include:
  - Cashback
  - Reward
  - Voucher
  - Smart Tag
  - Accelerated Reward
  - Partner Points
- Click **Continue**.
- The **Offer Scope** screen is displayed.

![Step 3 - Select the Outcome Type](/images/d.png)

---

### Step 4: Configure the Offer Scope

After selecting the **Outcome Type**, the **Offer Scope** screen is displayed.

Use this page to configure the basic details of the Transactional Offer and define where the offer will be applicable.

#### Basic Offer Details

Provide the following information:

- **Offer Name** – Enter a name to identify the offer.
- **Offer Description** – Enter a brief description of the offer.
- **Offer Duration** – Select the start and end date and time during which the offer will remain active.
- **Eligibility Level** – Select how the offer eligibility should be evaluated:
  - **Account Level** – Evaluates eligibility independently for each account.
  - **Customer Level** – Evaluates eligibility across all accounts belonging to the customer.

![Step 4 - Configure Basic Offer Details](/images/e.png)

#### Offer Applicability

Define where the offer should be applicable.

Provide the following information:

- **Issuer** – Select the issuer for which the offer will be available.
- **Program Type** – Select the applicable program type.
- **Apply to all current and future programs** – Enable this option if the offer should apply to all existing and future programs.
- **Program** – If the above option is disabled, select one or more applicable programs.

After reviewing the configured details, click **Continue** to proceed to the **Offer User Base** configuration.

![Step 4 - Configure Offer Applicability](/images/f.png)

---

### Step 5: Configure the Offer User Base

The **Offer User Base** page allows you to define which users are eligible to participate in the Transactional Offer.

Choose the appropriate user base type based on your campaign requirements.

#### Static Base

Select **Static Base** to use a fixed set of eligible users. Once the offer goes live, the user base remains unchanged.

After selecting **Static Base**, choose the eligibility criteria that will be used to identify the eligible users.

The following eligibility options are available: 

| Eligibility Option | Description |
|--------------------|-------------|
| **Smart Tag** | Identifies users using the configured Smart Tag. |
| **Accounts** | Defines the user base using account-related information. |
| **Switch Account Number** | Identifies users using the Switch Account Number. |
| **Birthdate** | Defines the user base based on the customer's birth date. |
| **Account Creation Date** | Defines the user base based on the account creation date.

![Step 5 - Select Static Base](/images/g.png)

Select the required eligibility option and configure the corresponding values.

![Step 5 - Configure Static Base Eligibility](/images/g.1.png)

After completing the configuration:

- Click **Estimate User Base** to calculate the estimated number of eligible users.
- Review the estimated user count.
- Click **Continue** to proceed to the **Transaction Eligibility** configuration.

---

### Step 6: Configure Transaction Eligibility 

After configuring the **Offer User Base**, the **Transaction Eligibility** screen is displayed.

Use this page to define the transaction conditions that must be satisfied for the offer to be applied.

#### Configure Transaction Conditions

Select the transaction attributes that should be used to evaluate transaction eligibility.

The following transaction attributes are available:

| Transaction Attribute | Description |
|-----------------------|-------------|
| **MCC** | Defines eligibility based on the Merchant Category Code (MCC). |
| **MID** | Defines eligibility based on the Merchant Identifier (MID). |
| **TID** | Defines eligibility based on the Terminal Identifier (TID). |
| **POS Entry Mode** | Defines eligibility based on the Point-of-Sale (POS) entry mode. |
| **Transaction Value** | Defines eligibility based on the transaction amount. |
| **Transaction Type** | Defines eligibility based on the transaction type. |
| **Sovereignty Indicator** | Defines eligibility based on the transaction sovereignty indicator. |
| **Transaction Code** | Defines eligibility based on the transaction code. |
| **Payer VPA** | Defines eligibility based on the payer Virtual Payment Address (VPA). |
| **Payee VPA** | Defines eligibility based on the payee Virtual Payment Address (VPA). |

![Step 6 - Configure Transaction Eligibility](/images/h.png)

After selecting the required transaction attribute:

- Select the appropriate comparison operator.
- Enter or select the required value.
- Add additional transaction conditions, if required.
- Select the transaction posting date on which the eligibility should be evaluated.
- Click **Continue** to proceed to the **Offer Calculation** configuration.

---

### Step 7: Configure the Offer Calculation

After configuring the **Transaction Eligibility**, the **Offer Calculation** screen is displayed.

Use this page to define how the reward should be calculated for every qualifying transaction. The configured calculation rule determines the outcome that eligible customers receive when they satisfy the transaction eligibility criteria.

#### Configure the Offer Calculation Rule

Select the appropriate **Offer Calculation Rule** from the dropdown based on the reward calculation method.

The following calculation rules are available:

| Calculation Rule | Description |
|------------------|-------------|
| **Flat** | Awards a fixed reward amount for every qualifying transaction. |
| **Percentage** | Awards a percentage of the qualifying transaction amount. |
| **Slab** | Awards different reward values based on predefined transaction value ranges. |

After selecting the required calculation rule:

- Configure the reward value.
- Review the calculated outcome before proceeding.

![Step 7 - Configure Offer Calculation](/images/i.png)

#### Skip Cashback Capping (Optional)

Use the **Skip Cashback Capping** section to exclude cashback capping at specific levels.

The following options are available:

| Cashback Capping Level | Description |
|-------------------------|-------------|
| **Program Level** | Ignores cashback caps configured at the program level. |
| **Transaction Level** | Ignores cashback caps configured for individual transactions. |
| **MCC** | Ignores cashback caps configured for specific Merchant Category Codes (MCC). |
| **MID** | Ignores cashback caps configured for specific Merchant Identifiers (MID). |
| **Currency Level** | Ignores cashback caps configured for specific currencies. |

#### Allow Reversal

Enable **Allow Reversal** if the configured reward should be reversed when a qualifying transaction is reversed or refunded.

After completing the configuration:

- Review the configured calculation settings.
- Click **Continue** to proceed to the **Budget** configuration.

---

### Step 8: Configure the Budget

After configuring the **Offer Calculation**, the **Budget** screen is displayed.

Use this page to allocate the budget for the offer and define the approval workflow required before the offer is published.

#### Configure the Budget

Provide the following information:

- **Set Budget** – Enter the total budget allocated for the offer.
- **Approver** – Select the user responsible for reviewing and approving the configured budget.

![Step 8 - Configure Budget](/images/j.png)

#### Advanced Settings

Expand **Advanced Settings** to configure additional budget-related parameters, if applicable.

After completing the configuration:

- Verify the configured budget and approver details.
- Click **Continue** to proceed to the **Usage Limits** configuration.

---

### Step 9: Configure Usage Limits

After configuring the **Budget**, the **Usage Limits** screen is displayed.

Use this page to define the maximum benefit that a customer can receive within a specified time period.

#### Configure Usage Limits

Provide the following information:

- **Duration** – Select the period during which the usage limit should be applied (for example, Daily, Monthly, Quarterly, or Annual).
- **Limit Type** – Select the type of limit to be enforced.
- **Limit Value** – Enter the maximum reward amount or usage value that a customer can avail during the selected period.

If multiple usage limits are required:

- Click **Add more**.
- Configure additional usage limits as required.

![Step 9 - Configure Usage Limits](/images/k.png)

After completing the configuration:

- Review the configured usage limits.
- Click **Continue** to proceed to the **Posting** configuration.

---

### Step 10: Configure Posting

After configuring the **Usage Limits**, the **Posting** screen is displayed.

Use this page to define when the offer benefit should be posted to the customer's account and specify the eligibility criteria for posting.

#### Enable Posting

Enable **Posting** to allow the configured benefit to be credited to the customer's account.

#### Configure Posting Eligibility

Select the eligibility criteria that must be satisfied before the benefit is posted.

For each posting eligibility condition:

- Select the required eligibility attribute.
- Configure the corresponding values.
- Specify whether the condition is **Allowed** or **Restricted**, as applicable.
- Click **Add more** to configure additional posting eligibility conditions.

#### Configure Offer Posting

Choose when the offer benefit should be posted.

The following posting options are available:

| Posting Option | Description |
|----------------|-------------|
| **Post Immediately** | Credits the benefit immediately after the configured posting conditions are satisfied. |
| **Schedule** | Credits the benefit at a scheduled date and time. |

![Step 10 - Configure Posting](/images/l.png)

#### Configure Narration

Provide the narration that should be recorded in the ledger during posting and reversal.

Configure the following fields:

- **Posting Narration** – Enter the message to be recorded when the benefit is successfully posted.
- **Reversal Narration** – Enter the message to be recorded if the posted benefit is reversed.

![Step 10 - Configure Narration](/images/l.1.png)

After completing the configuration:

- Review the configured posting settings.
- Click **Continue** to proceed to the **Display Elements** configuration.

---

### Step 11: Configure Display Elements

After configuring the **Posting**, the **Display Elements** screen is displayed.

Use this page to configure how the offer is presented to customers in the PWA. You can customize the Home page, Offer page, and Offer Details page to provide a consistent customer experience.

Use this page to customize how the offer is presented to customers in the PWA. Configure the display elements for both the **Home** page and the **Offer** page.

#### Configure the Home Page

The **Home** page allows you to define how the offer appears on the customer's home screen.

Provide the following information:

- **Enable Display** – Enable this option to display the offer in the customer's PWA.
- **Labels** – Select one or more labels to categorize the offer.
- **Banner Title** – Enter the title to be displayed on the offer banner.
- **Banner Description** – Enter a brief description of the offer.
- **Display Title** – Enter the title of the offer displayed to the customer.
- **Display Description** – Enter the detailed description of the offer.
- **Display Order** – Specify the display priority of the offer.
- **Display Color** – Select the primary color used for the offer card in the PWA.
- **Offer/Merchant Logo** – Upload the logo to be displayed with the offer.
- **Background Illustration** – Upload the background image displayed on the offer page.

![Step 11 - Configure Home Page](/images/m.png)
![Step 11 - Configure Offer Page](/images/m.1.png)

After completing the configuration:

- Review the configured display elements.
- Click **Continue** to proceed to the **Communication** configuration.

#### Configure the Offer Details Page

The **Offer Details** page allows you to configure the detailed information that customers see after opening an offer in the PWA.

#### Section Details

Use the **Section Details** editor to create and organize the content displayed on the offer details page.

Provide the following information:

- **Section Title** – Enter a title for the section.
- **Section Content** – Enter the content that describes the offer. Use the rich text editor to format the content as required.
- **Add Condition** – Click **Add Condition** to create additional sections for displaying more offer information.

#### Configure Call-to-Action (CTA)

Provide the following information:

- **Text on Button** – Enter the text to be displayed on the CTA button.
- **Redirection Link** – Enter the URL to which customers should be redirected when they click the CTA button.

#### Advanced Settings

Enable **Advanced Settings** to configure additional display options for the offer, if required.

![Step 11 - Configure Offer Details Page](/images/n.png)

![Step 11 - Configure Offer Details Settings](/images/n.1.png)

After completing the configuration:

- Review the configured display details.
- Click **Continue** to proceed to the **Communication** configuration.
