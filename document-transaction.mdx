## What is a Transactional Offer?

<Info>
A **Transactional Offer** rewards customers for completing a **single qualifying transaction** that satisfies the configured eligibility criteria. Each qualifying transaction is evaluated independently, and eligible customers receive the configured reward based on the selected outcome type.
</Info>

---

## Create a Transactional Offer

To create a new Transactional Offer, begin from the **Manage Offers** page.

### Step 1: Create a New Offer

On the **Manage Offers** page:

- Click **+ Create New Offer** located in the upper-right corner.
- From the available options, select **Single Offer**.
- The offer creation journey is displayed.

![Step 1 - Create a Transactional Offer](/images/a.png)

---

### Step 2: Select the Offer Type

After selecting **Single Offer**, the **Offer Type** screen is displayed.

On this screen:

- Browse the available offer types.
- Select **Transaction Offers**.
- Click **Continue**.
- The **Outcome Type** screen is displayed.

![Step 2 - Select the Offer Type](/images/c.png)

---

### Step 3: Select the Outcome Type

After clicking **Continue**, the **Outcome Type** screen is displayed.

The Outcome Type determines the reward that customers receive after completing a qualifying transaction.

On this screen:

- Browse the available outcome types.
- Select the required outcome type.
- Available options include:
  - Cashback
  - Reward
  - Voucher
  - Smart Tag
  - Accelerated Reward
  - Partner Points
- Click **Continue**.
- The **Offer Scope** screen is displayed.

![Step 3 - Select the Outcome Type](/images/d.png)

## Step 5: Configure the Offer User Base

The **Offer User Base** configuration determines which users are eligible for the Transactional Offer.

You can configure the user base using either a **Static Base** or a **Dynamic Config**, depending on how you want eligible users to be managed throughout the offer lifecycle.

---

### Static Base

Select **Static Base** if you want to use a fixed set of eligible users.

Once the offer goes live, no additional users are added to the offer. Only the users who satisfy the configured eligibility criteria before the offer goes live are included.

![Static Base](/images/g.png)

After selecting **Static Base**, configure the eligibility criteria that should be used to identify the eligible users.

The following eligibility options are available:

| Eligibility | Description |
|-------------|-------------|
| **Smart Tag** | Identifies users using the configured Smart Tag. |
| **Accounts** | Uses account-related information to identify eligible users. |
| **Switch Account Number** | Uses the Switch Account Number to identify eligible users. |
| **Birthdate** | Identifies users based on their birth date. |
| **Account Creation Date** | Identifies users based on the account creation date. |

After configuring the required eligibility criteria, click **Estimate User Base** to calculate the estimated number of eligible users.

---

### Dynamic Config

Select **Dynamic Config** if you want the user base to be updated dynamically.

Eligible users who satisfy the configured eligibility criteria during the offer period are automatically added until the configured end date.

![Dynamic Config](/images/g.1.png)

Configure the required eligibility criteria in the same way as the **Static Base** configuration.

After configuring the required eligibility criteria, click **Estimate User Base** to calculate the estimated number of eligible users.
