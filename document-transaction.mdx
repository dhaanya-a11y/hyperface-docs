## Introduction

This guide explains how to create and configure Transactional Offers in the Hyperface Dashboard.

Transactional Offers enable issuers to reward customers for performing qualifying individual transactions. Eligibility is configured using transaction-level conditions such as MCC, MID, Transaction Amount, POS Entry Mode, and other transaction attributes.

This document is intended for teams responsible for creating and managing offers within the Hyperface platform.

---

## What are Transactional Offers?

Transactional Offers are reward campaigns triggered by a customer completing a single qualifying transaction.

Unlike milestone offers, no spend accumulation is required. Reward eligibility is evaluated per transaction.

A Transactional Offer consists of:

- Offer Details
- User Base Configuration
- Transaction Eligibility Rules
- Offer Calculation Rules
- Usage Limits
- Posting Configuration
- Display Elements

## Navigating to the Offer Listing Page

2. Select **Manage Offers**.
3. The Offer Listing page will appear.

### Summary Counters

- Active Now

- Expiring This Week

### Filter and Search

Use the Filter Bar to narrow down offers by:

- Status
- Scope
- Outcome Type
- Offer Type
- Program
- Created By
- Start Date
- End Date

---

# Create a Transactional Offer

1. Navigate to the appropriate offer category.
2. Click **Create New Offer**.
3. Select **Transactional Offer**.
4. Click **Continue**.
5. Select an Outcome Type.
6. Continue to Offer Scope.

---

## Step 1: Configure Offer Details

Configure:

- Offer Name
- Offer Description
- Offer Duration
- Issuer
- Program Selection

### Required Fields

| Field | Required |
|---------|---------|
| Offer Name | Yes |
| Offer Description | Yes |
| Start Date / End Date | Yes |
| Eligibility Level | Yes |
| Issuer | Yes |
| Program | Yes |

---

## Step 2: Configure Offer User Base

Determine which customers can participate.

### User Base Types

#### Static Base

- Fixed customer list
- Smart Tag filtering supported

#### Dynamic Base

- Opt-In
- Smart Tag Attachment

### Enrollment Methods

- Opt-In
- Smart Tag Attachment

---

## Step 3: Configure Transaction Eligibility

This step defines transaction-level conditions.

Supported attributes:

- MCC
- MID
- TID
- Merchant Name
- Transaction Value
- Transaction Type
- POS Entry Mode
- Sovereignty Indicator
- Transaction Code

### Actions

- Allowed
- Denied

### Logic

- AND Conditions
- OR Groups

---

## Step 4: Configure Offer Calculation

Define how rewards are calculated.

### Reward Types

#### Fixed Reward

Flat reward per transaction.

#### Percentage Based Reward

Percentage of transaction amount.

#### Slab Based Reward

Reward based on transaction value brackets.

---

## Step 5: Configure Usage Limits

Optional reward earning limits.

### Available Limits

- Daily Limit
- Weekly Limit
- Monthly Limit
- Lifetime Limit
- Transaction Count Limit

---

## Step 6: Configure Posting

Defines when rewards are posted.

### Posting Methods

#### Post Immediately

Reward posted after validation.

#### Schedule

Reward posted after configured delay.

### Required Fields

- Enable Posting
- Narration
- Posting Method

Optional:

- Posting Eligibility
- Reversal Narration

## Step 8: Review and Publish

Review all configurations:

- Offer Details
- User Base
- Eligibility
- Calculation
- Usage Limits
- Posting
- Display Elements

Click **Publish**.

The offer will enter **Scheduled** status and become **Active** automatically at the configured start time.
