## What is a Transactional Offer?

<Info>
A **Transactional Offer** rewards customers for completing a **single qualifying transaction** that satisfies the configured eligibility criteria. Each qualifying transaction is evaluated independently, and eligible customers receive the configured reward based on the selected outcome type.
</Info>
  
---

## Create a Transactional Offer

To create a new Transactional Offer, begin from the **Manage Offers** page.

### Step 1: Create a New Offer

On the **Manage Offers** page:

- Click **+ Create New Offer** located in the upper-right corner.
- From the available options, select **Single Offer**.
- The offer creation journey is displayed.

![Step 1 - Create a Transactional Offer](/images/a.png)

---

### Step 2: Select the Offer Type

After selecting **Single Offer**, the **Offer Type** screen is displayed.

On this screen:

- Browse the available offer types.
- Select **Transaction Offers**.
- Click **Continue**.
- The **Outcome Type** screen is displayed.

![Step 2 - Select the Offer Type](/images/c.png)

---

### Step 3: Select the Outcome Type

After clicking **Continue**, the **Outcome Type** screen is displayed.

The Outcome Type determines the reward that customers receive after completing a qualifying transaction.

On this screen:

- Browse the available outcome types.
- Select the required outcome type.
- Available options include:
  - Cashback
  - Reward
  - Voucher
  - Smart Tag
  - Accelerated Reward
  - Partner Points
- Click **Continue**.
- The **Offer Scope** screen is displayed.

![Step 3 - Select the Outcome Type](/images/d.png)

---

### Step 4: Configure the Offer Scope

After selecting the **Outcome Type**, the **Offer Scope** screen is displayed.

Use this page to configure the basic details of the Transactional Offer and define where the offer will be applicable.

#### Basic Offer Details

Provide the following information:

- **Offer Name** – Enter a name to identify the offer.
- **Offer Description** – Enter a brief description of the offer.
- **Offer Duration** – Select the start and end date and time during which the offer will remain active.
- **Eligibility Level** – Select how the offer eligibility should be evaluated:
  - **Account Level** – Evaluates eligibility independently for each account.
  - **Customer Level** – Evaluates eligibility across all accounts belonging to the customer.

![Step 4 - Configure Basic Offer Details](/images/e.png)

#### Offer Applicability

Define where the offer should be applicable.

Provide the following information:

- **Issuer** – Select the issuer for which the offer will be available.
- **Program Type** – Select the applicable program type.
- **Apply to all current and future programs** – Enable this option if the offer should apply to all existing and future programs.
- **Program** – If the above option is disabled, select one or more applicable programs.

After reviewing the configured details, click **Continue** to proceed to the **Offer User Base** configuration.

![Step 4 - Configure Offer Applicability](/images/f.png)

---

### Step 5: Configure the Offer User Base

The **Offer User Base** page allows you to define which users are eligible to participate in the Transactional Offer.

Choose the appropriate user base type based on your campaign requirements.

#### Static Base

Select **Static Base** to use a fixed set of eligible users. Once the offer goes live, the user base remains unchanged.

After selecting **Static Base**, choose the eligibility criteria that will be used to identify the eligible users.

The following eligibility options are available: 

| Eligibility Option | Description |
|--------------------|-------------|
| **Smart Tag** | Identifies users using the configured Smart Tag. |
| **Accounts** | Defines the user base using account-related information. |
| **Switch Account Number** | Identifies users using the Switch Account Number. |
| **Birthdate** | Defines the user base based on the customer's birth date. |
| **Account Creation Date** | Defines the user base based on the account creation date.

![Step 5 - Select Static Base](/images/g.png)

Select the required eligibility option and configure the corresponding values.

![Step 5 - Configure Static Base Eligibility](/images/g.1.png)

After completing the configuration:

- Click **Estimate User Base** to calculate the estimated number of eligible users.
- Review the estimated user count.
- Click **Continue** to proceed to the **Transaction Eligibility** configuration.

---

### Step 6: Configure Transaction Eligibility 

After configuring the **Offer User Base**, the **Transaction Eligibility** screen is displayed.

Use this page to define the transaction conditions that must be satisfied for the offer to be applied.

#### Configure Transaction Conditions

Select the transaction attributes that should be used to evaluate transaction eligibility.

The following transaction attributes are available:

| Transaction Attribute | Description |
|-----------------------|-------------|
| **MCC** | Defines eligibility based on the Merchant Category Code (MCC). |
| **MID** | Defines eligibility based on the Merchant Identifier (MID). |
| **TID** | Defines eligibility based on the Terminal Identifier (TID). |
| **POS Entry Mode** | Defines eligibility based on the Point-of-Sale (POS) entry mode. |
| **Transaction Value** | Defines eligibility based on the transaction amount. |
| **Transaction Type** | Defines eligibility based on the transaction type. |
| **Sovereignty Indicator** | Defines eligibility based on the transaction sovereignty indicator. |
| **Transaction Code** | Defines eligibility based on the transaction code. |
| **Payer VPA** | Defines eligibility based on the payer Virtual Payment Address (VPA). |
| **Payee VPA** | Defines eligibility based on the payee Virtual Payment Address (VPA). |

![Step 6 - Configure Transaction Eligibility](/images/h.png)

After selecting the required transaction attribute:

- Select the appropriate comparison operator.
- Enter or select the required value.
- Add additional transaction conditions, if required.
- Select the transaction posting date on which the eligibility should be evaluated.
- Click **Continue** to proceed to the **Offer Calculation** configuration.

---
---

### Step 7: Configure Offer Calculation

After configuring the **Transaction Eligibility**, the **Offer Calculation** screen is displayed.

Use this page to define how the reward should be calculated for every qualifying transaction.

#### Configure the Offer Calculation Rule

Select the appropriate **Offer Calculation Rule** from the dropdown.

The following calculation rules are available:

| Calculation Rule | Description |
|------------------|-------------|
| **Flat** | Provides a fixed reward amount for every qualifying transaction. |
| **Percentage** | Calculates the reward as a percentage of the qualifying transaction amount. |
| **Slab** | Calculates the reward based on predefined transaction amount ranges. |

After selecting the required calculation rule, configure the corresponding reward value.

For example, when **Flat** is selected:

- Enter the cashback amount to be rewarded for each qualifying transaction.

![Step 7 - Configure Offer Calculation](/images/i.png)

#### Skip Cashback Capping (Optional)

Enable the required toggle to skip cashback capping at the following levels:

| Cashback Capping Level | Description |
|-------------------------|-------------|
| **Program Level** | Skips the cashback cap configured at the program level. |
| **Transaction Level** | Skips the cashback cap configured for individual transactions. |
| **MCC** | Skips cashback capping configured for specific Merchant Category Codes (MCC). |
| **MID** | Skips cashback capping configured for specific Merchant Identifiers (MID). |
| **Currency Level** | Skips cashback capping configured for specific currencies. |

#### Allow Reversal

Enable **Allow Reversal** if the cashback should be reversed when a qualifying transaction is reversed or refunded.

After completing the configuration:

- Review the configured calculation rule.
- Click **Continue** to proceed to the **Budget** configuration.

---

### Step 8: Configure Budget

After configuring the **Offer Calculation**, the **Budget** screen is displayed.

Use this page to define the budget allocated for the offer and configure the approval workflow.

#### Configure the Budget

Provide the following information:

- **Set Budget** – Enter the total budget allocated for the offer.
- **Approver** – Select the user responsible for approving the configured budget.

![Step 8 - Configure Budget](/images/j.png)

#### Advanced Settings (Optional)

Expand **Advanced Settings** to configure additional budget-related options, if required.

After completing the configuration:

- Verify the configured budget and approver details.
- Click **Continue** to proceed to the **Usage Limits** configuration.
