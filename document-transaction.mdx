## Introduction

This guide explains how to create and configure Transactional Offers in the Hyperface Dashboard.

Transactional Offers enable issuers to reward customers for performing qualifying individual transactions. Eligibility is configured using transaction-level conditions such as MCC, MID, Transaction Amount, POS Entry Mode, and other transaction attributes.

This document covers the complete offer creation journey, offer listing management, and key field-level guidance.

---

## What are Transactional Offers?
Transactional Offers are reward campaigns triggered when a customer completes a single qualifying transaction.

Unlike Milestone Offers, no spend accumulation is required. Reward eligibility is evaluated individually for each transaction.

## Navigating to the Offer Listing Page

The Offer Listing page is the central hub for viewing and managing all offers configured in the system.

To access the Offer Listing page:

1. Navigate to the **Offers** section from the left navigation menu.
2. Select **Manage Offers**.
3. The Offer Listing page will appear.

### Offer Category Tabs

Offers are grouped into categories displayed as tabs at the top of the listing page. Select the appropriate tab based on the offer type being configured before creating a new offer.

### Summary Counters

The Offer Listing page displays key metrics for quick monitoring:

- Active Now
- Approval Pending
- Revision Required
- Expiring This Week

### Filter and Search

Use the Filter Bar to narrow down offers by:

- Status
- Scope
- Outcome Type
- Offer Type
- Program
- Created By
- Start Date
- End Date

Use the Search Bar to locate offers using Offer Name or Offer ID.
### Components

A Transactional Offer consists of:

* Offer Details
* User Base Configuration
* Transaction Eligibility Rules
* Offer Calculation Rules
* Usage Limits

* Display Elements

## Create a Transactional Offer

Follow the steps below to create a new Transactional Offer.

1. Navigate to the Offer Listing page.
2. Select the appropriate Offer Category tab.
3. Click **Create New Offer**.
4. Select **Transactional Offer**.
5. Choose the Outcome Type.
6. Continue through the configuration journey.

Select the reward outcome that customers will receive after completing a qualifying transaction.

## Step 1: Configure Offer Details
### Important Notes

- Offer Name does not need to be unique.
- Offer Description is for internal reference only.
- Start time must be at least 5 minutes in the future.
- End date must be later than the start date.
- Offers are created in Scheduled status and automatically become Active at the configured start time.
- If "Apply to all programs" is enabled, individual program selection is disabled.

This section captures the basic identity and scope of the offer.

Configure:

* Offer Name
* Offer Description
* Start Date & Time
* End Date & Time
* Eligibility Level
* Issuer
* Program Selection

### Program Configuration

* Apply to all current and future programs

## Step 3: Configure Transaction Eligibility

### Eligibility Logic

All conditions within a group use AND logic.

A customer becomes eligible when a transaction satisfies all configured conditions within a group.

Conditions can be configured using:

- Allowed
- Denied
- List Based Inputs
- Direct Input Values

### Creating Lists

Lists can be created directly from the eligibility screen.

Examples include:

- Shopping MCCs
- Fuel MCCs
- Merchant MID Lists

Lists can be configured as Immutable or Editable depending on business requirements.

## Step 4: Configure Offer Calculation

### Calculation Examples

#### Fixed Reward

Every qualifying transaction earns a predefined reward amount.

Example:

- Qualifying supermarket transaction → 50 reward points

#### Percentage Based Reward

Reward is calculated as a percentage of transaction value.

Example:

- Spend ₹2,000
- Reward = 5%
- Earn 100 reward points

#### Slab Based Reward

Rewards vary based on transaction value ranges.

Example:

- ₹500–₹1,999 → 50 points
-- ₹2,000–₹4,999 → 150 points

## Step 6: Configure Posting
### Posting Recommendations

Scheduled Posting is recommended because it provides time for transaction reversals and refunds before rewards are issued.

Immediate Posting should only be used when reversal risk is minimal.

## Managing Existing Offers

The Offer Listing page allows users to manage existing offers throughout their lifecycle.

### Available Actions

- View
- Edit
- Duplicate
- Delete
- Expire

### Offer Statuses

#### Active

Offer is currently live and available to eligible customers.

#### Scheduled

Offer has been configured and will become active at a future date.

#### Draft

Offer has been created but not yet activated.

#### Expired

Offer has reached its end date and is no longer active.

### Viewing Offer Details

Click any offer name from the Offer Listing page to view:

- Eligibility Rules
- Calculation Rules
- Usage Limits
- Posting Configuration
- Display Configuration

### Important Notes

- Active offers can only be edited for display settings and end date.
- Eligibility and calculation rules are locked once the offer becomes active.
- Duplicated offers are created in Draft status.
- Scheduled offers can be deleted before activation.
### Viewing Offer Details

Click any offer name from the Offer Listing page to view the complete offer configuration, eligibility rules, calculation settings, usage limits, posting setup, and display configuration.

## Best Practices

* Verify eligibility conditions before publishing.
* Review reward calculations carefully.
* Configure posting delays to handle reversals and refunds.
* Validate display content before making the offer live.
* Test offer scenarios before activation.

## Notes

* Active offers have limited edit capabilities.
* Eligibility and calculation rules become locked once active.
* Scheduled offers can be modified before activation.
* Expired offers can be duplicated to create new campaigns.
