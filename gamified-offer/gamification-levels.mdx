## Step 4: Configure Gamification Levels

The Gamification Levels step is what distinguishes a Gamified Offer from a Milestone Offer. Instead of a single threshold, you configure a **ladder of thresholds**, each unlocking its own reward.

### Gamification Levels Best Practices

The level configuration determines how customer progress is tracked and when each reward is triggered.

Before publishing:

* Verify aggregation type selection.
* Validate threshold values for every level, in ascending order.
* Ensure reward value increases (or otherwise makes sense) as levels progress.
* Review subscription cycle configuration.
* Confirm transaction eligibility conditions apply consistently across all levels.
* Test level unlock scenarios, including customers who cross multiple thresholds in a single cycle.

### Example Transaction Eligibility Rule

Configuration:

* MCC = Grocery
* Transaction Value > ₹1,000
* Status = Allowed

Result:

Only grocery transactions above ₹1,000 contribute toward level progress.

### Aggregation Types

#### Sum of Transaction Values

Calculates the total eligible spend accumulated by the customer.

#### Count of Transactions

Calculates the total number of eligible transactions completed by the customer.

### Level Configuration

Each level requires:

- **Level Name** – Label used to identify the level internally (for example, Level 1, Silver, Gold).
- **Threshold Value** – The cumulative spend or transaction count required to unlock this level.
- **Threshold Operator** – Greater Than, Greater Than or Equal To, Less Than, or Equal To.
- **Reward Configuration** – The outcome value awarded when the level is unlocked (configured in Step 5).

Click **Add Level** to configure additional thresholds. Levels are evaluated in ascending order of threshold value.

### Threshold Configuration

Configure per level:

- Greater Than
- Greater Than or Equal To
- Less Than
- Equal To

### Subscription Cycle

Available options:

- Custom
- Billing Month

### Transaction Eligibility

Supported filters (applied across all levels):

- MCC
- MID
- Merchant Name
- Transaction Value
- Transaction Type
- POS Entry Mode

### Processing Date

Choose:

- Transaction Date
- Posting Date

### Level Behaviour

Lock Level Progress On Achievement determines whether cumulative progress continues to be tracked toward the next level once a level is unlocked.

- Enabled: Progress continues to accumulate so the customer can unlock subsequent levels within the same cycle.
- Disabled: Progress resets once a level is unlocked, requiring the cycle to restart before the next level can be pursued.

<Note>
In most Gamified Offer configurations, Lock Level Progress On Achievement should remain **Disabled**, since the purpose of the offer is to allow customers to keep climbing through multiple levels within a single cycle.
</Note>

### Subscription Cycle Examples

Example 1:

- Billing Month
- Number of Cycles = 1
- Level 1 = ₹5,000, Level 2 = ₹10,000, Level 3 = ₹20,000

Within one billing cycle, a customer spending ₹22,000 unlocks all three levels and receives all three rewards.

Example 2:

- Custom Duration = 30 Days
- Number of Cycles = 3
- Level 1 = 3 transactions, Level 2 = 7 transactions, Level 3 = 12 transactions

The level ladder resets every 30 days and can be climbed up to three times across the offer duration.
