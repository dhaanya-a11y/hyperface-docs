## Step 7: Configure Posting

### Posting Best Practices

Posting configuration determines when level rewards become visible to customers. Posting rules apply uniformly to rewards issued at every level.

Scheduled Posting is generally recommended because it provides sufficient time for transaction reversals, refunds, and settlement validation.

### Operational Recommendations

* Configure posting delays based on reversal windows.
* Use meaningful posting narrations that identify which level was unlocked.
* Verify reward expiry settings.
* Review reversal narration before publishing.
* Test reward posting behaviour for multiple levels unlocked within the same cycle.

### Posting Options

#### Post Immediately

Rewards are posted immediately after validation, as soon as a level is unlocked.

#### Scheduled Posting

Rewards are posted after a configured delay, once a level is unlocked.

### Configuration

- Enable Posting
- Posting Eligibility
- Days After
- Event
- Reward Point Expiry
- Posting Narration
- Reversal Narration

### Posting Recommendations

Scheduled Posting is recommended because it allows sufficient time for reversals and refunds before rewards are issued.

Immediate Posting should only be used when reversal risk is minimal.
