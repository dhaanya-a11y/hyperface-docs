## Overview

This guide explains how to configure the **Rent Cashback** offer in the Offer Management System (OMS).

## Business Objective

The **Rent Cashback** offer is designed to encourage customers to use their eligible credit cards for rent payments by rewarding qualifying transactions with cashback. Customers receive **5% cashback** on eligible rent transactions of **₹500 or more**, up to a maximum cashback of **₹500 per calendar month**.

# Basic Details

Navigate to **Offer Creation → Basic Details** and configure the following fields.

### Configuration

| Field | Value | Description |
|-------|-------|-------------|
| Offer Name | Rent Cashback Rewards | Name displayed for the offer. |
| Offer Description | Earn 5% cashback on eligible rent payments of ₹500 or more, up to a maximum cashback of ₹500 per calendar month. | Short description displayed for the offer. |
| Offer Duration | 03 Jul 2026 – 30 Dec 2026 | Period during which the offer remains active. |
| Eligibility Level | Account Level | Eligibility is evaluated independently for each eligible account. |
| Issuer | Demo-Bank | Credit card issuing bank. |
| Program Type | Credit Card | Product type for which the offer is applicable. |
| Apply to All Current and Future Programs | Disabled | The offer applies only to the selected program. |
| Program | IOB DEMO | Credit card program eligible for this offer. |

> **Selected Configuration:** The offer is configured at the **Customer Level**, allowing eligibility to be evaluated across all eligible accounts of a customer.

---

# Set Up Offer User Base

After configuring the basic details, define the customer group that will be eligible for the offer.

## User Base

The **User Base** determines whether the list of eligible customers remains fixed or is updated during the offer duration.

### Configuration

| Field | Value | Description |
|-------|-------|-------------|
| User Base | Static Base | The eligible customer list is fixed when the offer goes live. |

> **Selected Configuration:** Static Base ensures that only customers included before the offer goes live are eligible for the offer.

---

# Set Up Transaction Eligibility

Configure the transaction conditions that determine whether a transaction qualifies for cashback.

## Transaction Eligibility

### Configuration

| Field | Value | Description |
|-------|-------|-------------|
| Transaction Eligibility | Default | Uses the default transaction eligibility configuration. |
| MCC | Rent MCC | Only transactions belonging to the configured Rent MCC are eligible. |
| Transaction Value | Greater than or equal to ₹500 | Only rent transactions of ₹500 or above qualify for cashback. |

---

## Date for Offer Processing

Choose when the offer eligibility should be evaluated.

### Configuration

| Field | Value | Description |
|-------|-------|-------------|
| Offer Processing Date | Transaction Date | Cashback eligibility is evaluated using the transaction date. |

> **Selected Configuration:** Using the **Transaction Date** ensures cashback is calculated based on when the customer performs the eligible rent transaction.

---

# Calculate the Offer Outcome

After configuring the transaction eligibility, define how cashback is calculated.

## Offer Calculation Rule

### Configuration

| Field | Value | Description |
|-------|-------|-------------|
| Offer Calculation Rule | Percentage with Max | Cashback is calculated as a percentage with a maximum monthly limit. |
| Cashback Percentage | 5% | Customers receive 5% cashback on eligible transactions. |
| Maximum Cashback | ₹500 | Maximum cashback allowed per calendar month. |

> **Selected Configuration:** Customers receive **5% cashback** on every eligible rent transaction, up to **₹500 cashback per calendar month**.

# Usage Limit

The **Usage Limit** defines the maximum cashback a customer can receive during the configured validity period.

### Configuration

| Field | Value | Description |
|-------|-------|-------------|
| Limit Duration | Month | Cashback limit resets every month. |
| Limit Type | Amount | Cashback is limited by amount. |
| Limit Value | ₹500 | Maximum cashback allowed every month. |

> **Selected Configuration:** Customers can receive up to **₹500 cashback per calendar month**.

---

# Define When to Post Your Offer

Configure how and when cashback is posted to the customer's account.

## Posting Eligibility

### Configuration

| Field | Value | Description |
|-------|-------|-------------|
| Enable Posting | Enabled | Allows cashback to be credited. |
| Posting Eligibility | Account Status | Cashback is posted only to eligible accounts. |
| Account Status | ACTIVE | Only active accounts receive cashback. |
| Offer Posting | Immediately | Cashback is credited immediately after an eligible transaction. |

# Craft the Perfect UI: Details Page

The **Details Page** provides customers with complete information about the offer.

## Section Details

### How It Works

1. Pay your rent using your eligible card.
2. Spend ₹500 or more on a rent payment.
3. Earn 5% cashback on eligible transactions.
4. Receive up to ₹500 cashback every calendar month.

### Terms & Conditions

- Valid only on eligible Rent MCC transactions.
- Minimum transaction amount: ₹500.
- Maximum cashback: ₹500 per calendar month.
- Cancelled or refunded transactions are not eligible.
- Standard terms and conditions apply.

### CTA Button

| Field | Value | Description |
|-------|-------|-------------|
| Text on Button | Pay Rent Now | CTA button displayed on the offer details page. |

> **Selected Configuration:** The details page clearly explains the offer benefits, eligibility criteria, and cashback limits, while the **"Pay Rent Now"** CTA encourages customers to make eligible rent payments.
