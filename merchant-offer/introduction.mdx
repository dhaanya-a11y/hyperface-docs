# Merchant Offer

## What is a Merchant Offer?

<Info>
A **Merchant Offer** rewards customers for making qualifying purchases at participating merchants. Each qualifying transaction is evaluated against the configured merchant eligibility criteria, and eligible customers receive the configured reward based on the selected outcome type.
</Info>

### Key Characteristics

- Merchant-specific eligibility
- Single-transaction evaluation
- Configurable outcome
- Flexible customer targeting
- Usage limits still apply

The **Offer Type** is the foundation of the offer configuration journey. It determines the criteria under which customers become eligible to receive rewards.

For this guide, select **Merchant Offer**, which is designed to reward customers for making qualifying purchases at participating merchants that satisfy the configured merchant eligibility criteria. This offer type is commonly used for merchant-specific cashback, reward points, and promotional campaigns.

## Configure Offer Details

![Step 1 - Configure Offer Details](/images/2.png)

The **Offer Details** define the basic information for the Merchant Offer and specify where the offer will be applicable.

Configure the following:

- **Offer Name** – Enter a unique name for the offer.
- **Offer Description** – Provide a brief description of the offer.
- **Offer Duration** – Select the offer validity period.
- **Eligibility Level** – Choose whether the offer should be evaluated at the **Account Level** or **Customer Level**.

![Step 1 - Configure Offer Applicability](/images/2.2.png)

Configure the offer applicability:

- **Issuer** – Select the issuer.
- **Program Type** – Select the applicable program type.
- **Apply to all current and future programs** – Apply the offer across all existing and future programs.
- **Program** – Select specific program(s) if required.

### Example

Create a Merchant Offer by entering the offer details, selecting the eligibility level, and choosing the applicable issuer and program.

## Configure Offer User Base

![Step 2 - Configure Offer User Base](/images/3.png)

Configure the **Offer User Base** to define which users are eligible for the Merchant Offer.

Configure the following:

- **Static Base** – Uses a fixed list of eligible users.
- **Dynamic Config** – Continuously evaluates users based on the configured eligibility criteria.
- **Eligibility** – Select the required eligibility criteria.
- **Estimate User Base** – Calculate the estimated number of eligible users.

### Example

Select the required user base type, configure the eligibility criteria, and use **Estimate User Base** to verify the expected audience.
