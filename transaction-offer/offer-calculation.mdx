### Step 7: Configure the Offer Calculation

After configuring the **Transaction Eligibility**, the **Offer Calculation** screen is displayed.

Use this page to define how the reward should be calculated for every qualifying transaction. The configured calculation rule determines the outcome that eligible customers receive when they satisfy the transaction eligibility criteria.

#### Configure the Offer Calculation Rule

Select the appropriate **Offer Calculation Rule** from the dropdown based on the reward calculation method.

The following calculation rules are available:

| Calculation Rule | Description |
|------------------|-------------|
| **Flat** | Awards a fixed reward amount for every qualifying transaction. |
| **Percentage** | Awards a percentage of the qualifying transaction amount. |
| **Slab** | Awards different reward values based on predefined transaction value ranges. |

After selecting the required calculation rule:

- Configure the reward value.
- Review the calculated outcome before proceeding.

#### Skip Cashback Capping (Optional)

Use the **Skip Cashback Capping** section to exclude cashback capping at specific levels.

The following options are available:

| Cashback Capping Level | Description |
|-------------------------|-------------|
| **Program Level** | Ignores cashback caps configured at the program level. |
| **Transaction Level** | Ignores cashback caps configured for individual transactions. |
| **MCC** | Ignores cashback caps configured for specific Merchant Category Codes (MCC). |
| **MID** | Ignores cashback caps configured for specific Merchant Identifiers (MID). |
| **Currency Level** | Ignores cashback caps configured for specific currencies. |

#### Allow Reversal

Enable **Allow Reversal** if the configured reward should be reversed when a qualifying transaction is reversed or refunded.

After completing the configuration:

- Review the configured calculation settings.
- Click **Continue** to proceed to the **Budget** configuration.
