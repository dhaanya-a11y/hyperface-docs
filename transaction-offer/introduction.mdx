# Transactional Offer

## What is a Transactional Offer?

<Info>
A **Transactional Offer** rewards customers for completing a **single qualifying transaction** that satisfies the configured eligibility criteria. Each qualifying transaction is evaluated independently, and eligible customers receive the configured reward based on the selected outcome type.
</Info>

### Key characteristics

- **Single-transaction evaluation** – Each transaction is checked independently against the eligibility criteria; there is no requirement to accumulate spend or transaction count.
- **Immediate qualification** – A customer becomes eligible for the reward the moment a single transaction satisfies the configured conditions.
- **Configurable outcome** – The reward can be issued as Cashback, Reward Points, a Voucher, a Smart Tag, an Accelerated Reward, or Partner Points.
- **Usage limits still apply** – While each transaction is evaluated on its own, the total benefit a customer can earn is still bounded by the configured **Usage Limits** (for example, a monthly cashback cap).

The **Offer Type** is the foundation of the offer configuration journey. It determines the criteria under which customers become eligible to receive rewards.

For this guide, select **Transaction Offers**, which is designed to reward customers based on qualifying transactions that meet the configured eligibility rules. This offer type is commonly used for campaigns such as cashback, discounts, and reward points.

## Configure the Outcome Type

![Step 3 - Select the Outcome Type](/images/d.png)

The **Outcome Type** defines the reward that customers receive after successfully meeting the configured offer conditions. Select the outcome that aligns with your campaign objective.

For this walkthrough, select **Cashback** as the outcome type. Cashback rewards customers by crediting a predefined amount or percentage once an eligible transaction satisfies all configured offer criteria.

### Example

Suppose you want to reward customers with **₹100 cashback** after they make a qualifying purchase of **₹1,000 or more**. In this case, select **Cashback** as the outcome type and configure the reward value in the subsequent steps.

After selecting the required **Outcome Type**, click **Continue** to proceed to the **Offer Scope** configuration.

## Configure the Offer Scope

![Step 4 - Configure Basic Offer Details](/images/e.png)

The **Offer Scope** defines where the Transactional Offer will be applicable. In this step, configure the basic offer details and specify the issuer and program(s) for which the offer should be available.

Configure the following:

- **Offer Name** – Enter a unique name for the offer.
- **Offer Description** – Provide a brief description of the offer.
- **Offer Duration** – Select the offer validity period.
- **Eligibility Level** – Choose whether the offer should be evaluated at the **Account Level** or **Customer Level**.

![Step 4 - Configure Offer Applicability](/images/f.png)

Define the applicability of the offer by configuring:

- **Issuer** – Select the issuer.
- **Program Type** – Select the applicable product category.
- **Apply to all current and future programs** – Enable this option to apply the offer across all existing and future programs.
- **Program** – Select specific program(s) if the offer should not apply to all programs.

### Example

Create a Transactional Offer by entering the offer details, selecting the required eligibility level, and choosing the applicable issuer and program(s). Enable **Apply to all current and future programs** if the offer should automatically apply to every program under the selected program type.

## Configure Offer User Base

![Step 5 - Select Static Base](/images/g.png)

Configure the **Offer User Base** to define which users are eligible for the Transactional Offer.

First, select the type of user base:

- **Static Base** – Uses a fixed list of eligible users that remains unchanged after the offer goes live.
- **Dynamic Config** – Continuously evaluates users against the configured eligibility criteria, allowing new users to become eligible over time.

Next, define the eligibility criteria for the selected user base.

Configure the following:

- **Eligibility** – Select the required user group or account scope.
- **Add Criteria** – Add one or more conditions to further refine the eligible user base.
- **Estimate User Base** – Calculate the estimated number of users that satisfy the configured criteria before publishing the offer.

### Example

Suppose the offer should be available only to a specific group of eligible users. Select the appropriate user base type, configure the required eligibility criteria, add additional filters if needed, and use **Estimate User Base** to verify the expected audience before proceeding.

## Configure Offer Calculation

![Step 7 - Configure Offer Calculation](/images/i.png)

Configure how the offer outcome should be calculated for eligible transactions.

Configure the following:

- **Offer Calculation Rule** – Select the calculation method for the offer outcome.
- **Outcome Value** – Specify the reward amount or value to be awarded to the customer.
- **Skip Cashback Capping** – Enable or disable cashback capping at the required level, such as **Program**, **Transaction**, **MCC**, **MID**, or **Currency**.
- **Allow Reversal** – Enable this option to reverse the reward if the qualifying transaction is reversed.

### Example

Select the required calculation rule, configure the reward value, choose the applicable cashback capping settings, and enable **Allow Reversal** if rewards should be reversed for reversed transactions.

The **Transaction Eligibility** step defines the conditions that determine whether a transaction qualifies for the Transactional Offer. Every transaction made by an eligible customer is evaluated against the configured transaction rules, and rewards are issued only when all the specified conditions are satisfied.

Configure one or more transaction attributes by selecting the required comparison operators and corresponding values. Multiple eligibility conditions can be combined to precisely identify qualifying transactions.

### Configure Transaction Conditions

Build the transaction eligibility rules by configuring the following:

- **Transaction Attribute** – Select the transaction attribute that should be evaluated.
- **Comparison Operator** – Choose the appropriate operator to compare the selected attribute.
- **Value** – Specify the value against which the transaction attribute will be evaluated.
- **Add Criteria** – Add multiple conditions to create more granular transaction eligibility rules.
- **Transaction Posting Date** – Select the transaction posting date that should be considered while evaluating transaction eligibility.

### Supported Transaction Attributes

You can configure eligibility using supported transaction attributes such as:

- **Merchant Category Code (MCC)**
- **Merchant ID (MID)**
- **Terminal ID (TID)**
- **POS Entry Mode**
- **Transaction Value**
- **Transaction Type**
- **Sovereignty Indicator**
- **Transaction Code**
- **Payer VPA**
- **Payee VPA**

### Example

Suppose you're creating a **5% Fuel Cashback Offer** that should only apply to fuel purchases above **₹2,000**.

Configure the **Merchant Category Code (MCC)** to identify fuel merchants, set the **Transaction Value** to **Greater than or equal to ₹2,000**, and add any additional conditions required for your campaign.

<Note>

The **Offer User Base** determines **who** is eligible for the offer, while **Transaction Eligibility** determines **which transactions** made by those customers qualify for rewards.

</Note>

The **Offer Calculation** step defines how rewards are calculated for transactions that satisfy the configured eligibility criteria. Select the appropriate calculation rule, configure the reward value, and define any additional reward processing settings before proceeding.

### Configure Offer Calculation

Configure the following:

- **Offer Calculation Rule** – Select the method used to calculate the offer outcome.
- **Outcome Value** – Specify the reward amount or value that will be awarded to eligible customers.
- **Skip Cashback Capping** – Enable or disable cashback capping at the required level, such as **Program**, **Transaction**, **Merchant Category Code (MCC)**, **Merchant ID (MID)**, or **Currency**.
- **Allow Reversal** – Enable this option to reverse the reward if the qualifying transaction is subsequently reversed.

### Example

Suppose you're creating a **5% Fuel Cashback Offer** with a maximum cashback of **₹200** per qualifying transaction.

Select the appropriate **Offer Calculation Rule**, configure the cashback percentage as the **Outcome Value**, define the required cashback capping settings, and enable **Allow Reversal** if the cashback should be reversed when the original transaction is reversed.

<Note>

Offer Calculation determines **how much reward** is issued after a transaction satisfies all configured eligibility conditions.

</Note>
